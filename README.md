# 🎮 游戏订阅提醒插件

> AstrBot 插件 · 结合 OneBot11 在 QQ 群中使用

订阅游戏发售日期与版本更新，插件会在每日指定时间自动通过 LLM 批量检索游戏信息，并在发售前 7/3/1 天及当天自动 @提醒订阅用户。

---

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 🕹️ 发售订阅 | 订阅游戏发售日期，发售前自动提醒 |
| 🔄 更新订阅 | 订阅游戏版本更新，更新当天自动提醒 |
| 📋 订阅列表 | 查看本群所有订阅（按发售/更新分类，渲染为图片） |
| 🗑️ 移除订阅 | 移除特定游戏的所有订阅 |
| 🧪 测试模式 | 带延时参数，快速验证检索结果 |
| 🧹 自动清理 | 发售日过后自动清除发售订阅（更新订阅长期有效） |
| ⚡ 批量检索 | 所有订阅游戏合并为一次 LLM 请求，节省 API 调用 |

---

## 📦 安装方式

### 方式一：AstrBot 插件市场（推荐）

在 AstrBot WebUI 的插件市场中搜索 `astrbot_plugin_game_subscription`，点击安装即可。

### 方式二：手动安装

```bash
# 将插件克隆到 AstrBot 的 data/plugins/ 目录下
cd AstrBot/data/plugins
git clone https://github.com/Nelson-Zeng/GAME_SUB.git astrbot_plugin_game_subscription
```

安装完成后，在 AstrBot WebUI 中重载插件或重启 AstrBot。

## 🚀 指令说明

所有指令需在 **QQ 群内** 使用。

### `/订阅游戏 <游戏名>`

订阅某游戏的发售日期提醒。

```
用户：/订阅游戏 黑神话悟空
机器人：✅ 已订阅《黑神话悟空》的发售提醒！
       📅 将在发售前 7, 3, 1 天及发售当天提醒您
```

### `/订阅游戏 <游戏名> <延时秒数>`

测试模式：订阅后延时执行一次检索，输出该游戏的预计发售日期。

```
用户：/订阅游戏 黑神话悟空 10
机器人：✅ 已订阅《黑神话悟空》发售提醒（测试模式）
       ⏱️ 将在 10 秒后执行检索并输出结果...

（10秒后）
机器人：@用户 🧪 测试检索结果
       🎮 黑神话悟空
       📅 预计发售日期: 2024-08-20
       📌 状态: 已发售
       📦 已于 294 天前发售
```

### `/订阅更新 <游戏名>`

订阅某游戏的版本更新提醒，更新当天自动通知。

```
用户：/订阅更新 原神
机器人：✅ 已订阅《原神》的更新提醒！
       📅 游戏更新当天将自动提醒您
```

### `/订阅列表`

查看当前群内所有订阅，按游戏发售/更新分类显示（渲染为图片）。

```
用户：/订阅列表
机器人：[图片：游戏订阅列表，含发售订阅和更新订阅两个分区]
```

### `/移除订阅 <游戏名>`

移除本群对特定游戏的所有订阅（发售 + 更新）。

```
用户：/移除订阅 原神
机器人：✅ 已移除本群对《原神》的所有订阅（共 2 条）
```

## ⚙️ 配置项

在 AstrBot WebUI → 插件 → 游戏订阅提醒 中进行配置。

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `check_time.hour` | 整数 | `9` | 每日检索执行小时（24小时制，0-23） |
| `check_time.minute` | 整数 | `0` | 每日检索执行分钟（0-59） |
| `reminder_days` | 字符串 | `7,3,1,0` | 发售前提醒天数，逗号分隔，0 表示发售当天 |
| `search_provider_id` | 字符串 | 留空 | 指定 LLM 提供商 ID，留空则使用当前默认提供商 |

## 🔔 自动提醒逻辑

```
每日 09:00（可配置）
    │
    ├─ 收集所有「发售订阅」的游戏名
    │   └─ 一次性 LLM 批量查询发售日期
    │       └─ 若距今 ∈ {7, 3, 1, 0} 天
    │           ├─ @订阅用户 发送提醒
    │           └─ 若为发售当天 → 自动清除该游戏的发售订阅
    │
    └─ 收集所有「更新订阅」的游戏名
        └─ 一次性 LLM 批量查询更新日期
            └─ 若更新日期 == 今天
                └─ @订阅用户 发送更新提醒
                （更新订阅不会被清除，长期有效）
```

## 📂 数据存储

订阅数据持久化在 AstrBot 根目录的 `data/astrbot_plugin_game_subscription/subscriptions.json`，格式如下：

```json
{
  "release_subscriptions": {
    "游戏名": [
      {
        "user_id": "QQ号",
        "group_id": "群号",
        "unified_msg_origin": "会话标识",
        "subscribe_date": "2025-01-01"
      }
    ]
  },
  "update_subscriptions": {
    "游戏名": [...]
  }
}
```

## ⚠️ 注意事项

- 插件依赖 AstrBot 已配置的 **LLM 提供商** 进行游戏信息检索，请确保已配置至少一个可用的 LLM 提供商。
- `search_provider_id` 可用于指定独立的 LLM 提供商（如成本更低的模型），避免使用主对话模型。
- 同一用户在同一群内对同一游戏只能订阅一次（防重复）。
- 发售订阅在游戏发售当天提醒后自动清除；更新订阅长期有效，需手动移除。
- 插件仅支持在 **QQ 群内** 使用（需配合 OneBot11 协议端，如 NapCat、Lagrange 等）。
